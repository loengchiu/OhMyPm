# 评审稿写法参考

本文件只做评审稿写法参考，不替代 `skills/omp-review/SKILL.md` 中的执行规则。

## 1. 目标写法

- 评审稿直接写问题和结论，不写流程说明
- 对人可见内容优先用页面名、页面编号和自然语言描述
- 统一归并为事实问题、风险问题、建议问题
- 统一结论必须清楚，不靠语气暗示

## 2. 人类写法 vs AI 写法

### 更像人类的写法

- 先说问题，再说影响，再说建议
- 阻断项和建议项分开
- 结论短、硬、清楚

### 容易暴露 AI 味的写法

- 大段解释怎么评审出来的
- 把方法论写进评审结论
- 问题、风险、建议混成一段
- 没有明确通过 / 不通过 / 需补充

## 3. 问题写法

推荐顺序：

- 事实：现在缺什么、错什么、漏什么
- 风险：会造成什么影响
- 建议：下一步怎么处理

## 4. 结论写法

- 允许：当前可进入下一阶段
- 不允许：当前存在阻断，必须先修
- 需补充：当前不阻断主线，但必须补齐指定项
