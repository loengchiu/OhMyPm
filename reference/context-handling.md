# 上下文处理参考

本文件只做上下文处理参考，不替代 `contracts/context-guard.md` 中的硬规则。

## 1. 输入分块示例

### 长材料

- 先抽结构化摘要
- 再抽关键事实、待确认项、原文位置
- 当前动作只读本轮判断必需片段

### 高成本文件

- PDF / Word / 扫描件优先做摘要包
- 不直接整篇载入

## 2. 输出分块示例

### 回应稿

- 先给本轮结论
- 再按事项分块
- 待确认项保留 `A：` 填写位

### 原型

- 按页面分块

### PRD

- 按章节、模块或页面单元分块

### 评审材料

- 按变化点、风险点、待决策点分块

## 3. `context_risk.decision` 示例

- `当前未命中上下文防爆，按常规链路继续`
- `命中长度信号，先做摘要包后继续`
- `长度与复杂度同时升高，先冻结当前状态再继续`

## 4. 什么时候要冻结

- 对齐超过 6 轮
- 已确认事实、未确认内容、下一步建议开始混杂
- 当前动作需要同时依赖多轮历史和新增材料
