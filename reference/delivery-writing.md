# 正式交付写法参考

本文件只做正式交付写法参考，不替代 `contracts/delivery.md` 中的硬约束。

## 1. 原型怎么表达

- 原型先交付页面本体，再补标注解释
- 页面像真实系统页面，不像讲解海报
- 标注编号贴元素走，点击后再看解释
- 不用大 hero、大导语、大流程说明区替代页面主体
- 简单页轻写，复杂页重写

## 2. PRD 怎么表达

- PRD 是正式归档正文，不是原型复述稿
- 原型已经讲清的页面结构，不整段重写
- PRD 重点补规则、边界、权限、数据字典、数据影响、验收
- 章节结构稳定，正文按复杂度展开

## 3. 原型与 PRD 的分工示例

### 原型负责

- 页面落点
- 主流程
- 页面里有什么
- 从哪里进
- 点哪里
- 会发生什么

### PRD 负责

- 为什么这么判定
- 异常怎么走
- 权限怎么卡
- 字段和枚举怎么定义
- 数据怎么落
- 验收怎么判断

## 4. 原型好的样子

- 研发先看原型，能快速理解 80% 的页面与流程
- 页面主体占主舞台
- 标注不抢戏
- 页面切换、弹窗、关键状态变化都能看见

## 5. PRD 好的样子

- 单独归档也能成立
- 不依赖讲解口头补充
- 不依赖原型才能理解规则
- 研发和测试能直接据此实施与验收
