# OhMyPm 外部知识协同

## 1. 核心原则

系统知识不在 OhMyPm 内部做成全量知识库。

知识主仓放在外部，例如：

- Obsidian
- Markdown 笔记
- 附件、截图、旧文档、会议记录

OhMyPm 在需求处理中按需读取这些资料。

## 2. 三层结构

### 2.1 外部原始信息层

自然记录和收集的信息，例如：

- 系统简介
- 页面截图
- 操作手册
- 旧 PRD
- 临时笔记
- 会议结论

### 2.2 AI 整理层

在某次需求涉及某系统时：

- 由 AI 读取相关资料
- 去重
- 提炼
- 归纳
- 标记不确定项

### 2.3 可回写记忆层

AI 产出 Markdown 形式的系统记忆卡，并回写到外部系统供人工：

- 阅读
- 修改
- 增补
- 复用

## 3. 内部状态映射

项目内部状态中保留：

- `系统记忆引用`
- `新增资料记录`

含义分别是：

- 当前需求实际引用了哪些系统记忆卡
- 当前轮新增进入判断的资料有哪些

## 4. 工作原则

- 不要求预先喂完全部存量系统
- 原始资料天然允许零散和非结构化
- AI 只在当前需求需要时读取相关资料
- 整理结果以 Markdown 系统记忆卡形式回写
