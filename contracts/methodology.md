# OhMyPm 方法摘要

## 1. 对外交互硬规则

- 默认自然语言是主入口
- 命令只作为调试、强制指定和高级用户入口
- 每次对外输出最后，必须只保留一个“下一步唯一动作”
- 若当前存在待确认项或 PM 决策缺口，输出必须收口为：
  - `现在只需要你回答的唯一问题是：...`
- 若当前不存在待确认项，输出必须收口为：
  - `下一步：...`
- 一次只推进一个动作
- 若当前动作被阻断，不允许同时给多个动作菜单

## 2. 最小读取硬规则

- 先读状态，再判动作，再补最少规则
- 默认只读当前动作对应的一个 skill
- 默认只读当前动作必需的少量规则
- 长材料和外部知识只做局部回查
- 长文生成后只保留摘要、索引和稳定路径

## 3. 固定分析顺序

遇到真实需求时，优先按以下顺序思考：

1. 先判当前要做哪个固定动作
2. 再判范围
3. 再判结构
4. 再判表达载体
5. 最后判估算和推进条件

## 4. 总约束

- 不得先写产物再倒推任务
- 不得先写细节再补结构
- 不得用模板填空代替边界判断
- 不得把未确认范围包装成已确认范围
- 不得把结构性变化包装成普通补充
