# OhMyPm 分层加载规则

## 1. 目标

分层加载不是建议，而是运行时硬规则。

目标：

- 先用最小上下文判断当前动作
- 再按动作只读一个 skill
- 再按动作只读必要规则
- 长材料和外部知识只做局部回查
- 长文生成后只保留摘要、索引和稳定路径

## 2. 五层结构

### 第一层：入口层

本层只负责：

- 判断当前请求是不是 PM 协作请求
- 判断当前主控权属于 `OhMyPm` 还是 `ShitPM`
- 判断当前属于真实项目协作、机制验证样例，还是 demo/smoke 回放
- 判断当前更像哪一个固定动作

本层禁止：

- 直接读取交付规则
- 直接读取长材料
- 默认预读多个 skill
- 把样例里的业务口径问题当成真实项目问题推进

### 第二层：状态层

默认只读：

- `.ohmypm/status.json`
- `.ohmypm/memory.md` 的最小必要摘要

本层只回答：

- 当前大致走到哪一步
- 当前是否有阻塞
- 当前是否已有稳定交付物
- 当前是否处于真实协作还是样例场景

本层禁止：

- 整篇载入项目记忆
- 读取系统记忆整卡
- 读取外部知识原文

### 第三层：决策层

本层只做两件事：

- 读取当前动作对应的一个 skill
- 读取当前动作必须的少量规则

固定动作与实现映射：

- 听需求 -> `omp-listen`
- 先回应 -> `omp-reply`
- 对齐 -> `omp-align`
- 提唯一问题 -> `omp-check`
- 开工检查 -> `omp-ready`
- 做原型 -> `omp-proto`
- 写 PRD -> `omp-prd`
- 评审 -> `omp-review`
- 改需求 -> `omp-change`
- 修问题 -> `omp-fix`

最小规则读取：

- 涉及长材料或长输出时补读 `contracts/context-guard.md`
- 需要提问时补读 `contracts/ask-back.md`
- 开工检查或变更判断时补读 `contracts/gates.md`
- 进入正式交付时补读 `contracts/delivery.md`
- 写 PRD 详细需求说明时补读 `contracts/anchors.md`
- 做评审时补读 `contracts/review.md`
- 涉及复写时补读 `contracts/overwrite.md`
- 涉及记忆时补读 `contracts/memory.md`
- 涉及外部知识时补读 `contracts/knowledge.md`

本层禁止：

- 默认同时读取多个 skill
- 为了保险一次读很多规则
- 让 PM 自己决定该走哪个命令

### 第四层：交付层

只有在进入重动作时才进入本层：

- 做原型
- 写 PRD
- 评审
- 改需求
- 修问题

本层允许：

- 读取稳定基线
- 局部回查相关材料
- 引用已生成交付物

本层禁止：

- 整篇整包载入外部知识
- 一次性把多个长文都塞进当前上下文

### 第五层：归档层

本层只负责：

- 将产物写入稳定路径
- 回写摘要、索引、状态
- 记录下一步唯一动作或唯一问题

本层禁止：

- 把长文全文继续挂在活跃上下文里
- 把历史长文不断堆回后续动作

## 3. 对外交互规则

- 对外默认表现为一个会自己判断下一步的协作助手
- 用户默认只用自然语言
- 命令流只作为调试入口、强制入口和高级入口
- 每次输出最后只能收口为：
  - `现在建议你做的下一步是：...`
  - `现在只需要你回答的唯一问题是：...`

禁止：

- 给 PM 一串菜单自己选
- 直接把内部状态字段丢给 PM
- 让 PM 通过 usage 或 runbook 决定下一步
