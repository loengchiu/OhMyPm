# OhMyPm 边界守护契约

## 1. 作用

本契约只做一件事：

- 把当前分散在各处的越界防护统一收口

它优先解决四类问题：

- 样例污染真实
- 未确认伪装已确认
- 脱锚扩写
- 元话语污染正式产物

## 2. 四条总规则

### 2.1 样例不污染真实

- 必须先判断当前是 `真实项目协作` 还是 `机制验证样例 / demo`
- 样例中的业务参数、规则口径、角色映射不得当作真实项目事实继承
- 样例中的业务缺口不得抛给 PM 做真实业务回答

样例允许的处理方式只有：

- 占位值
- 内部修正
- 显式标注“仅用于机制验证”

### 2.2 未确认不伪装已确认

- 未确认事实不得写入 `confirmed_facts`
- 未确认规则不得写成确定性 PRD 正文
- 未确认范围不得包装成当前稳定范围
- 若缺口会推翻当前主判断，必须保留在 `open_questions` 或 `pending_confirmations`

### 2.3 缺锚点不扩写

- 没有模块锚点，不写模块正文
- 没有页面锚点，不写页面说明
- 没有流程 / 动作锚点，不写规则和动作正文
- 原型和 PRD 若不共用同一套核心锚点，不得伪装为同版本交付物

### 2.4 元话语不进正式产物

正式原型和 PRD 中不得出现：

- `模块锚点 / 页面锚点 / 动作锚点`
- `追溯元数据`
- `契约承接`
- `当前模板试跑`
- `当前规则用于机制验证`
- 其他 OMP 自己的方法论说明

这些内容只允许进入：

- 状态
- 项目记忆
- review pack
- 内部承接材料

## 3. 动作级约束

### 3.1 听需求 / 回应

- 不得在上下文包缺失时伪装成“已理解”
- 不得把系统未知、页面未知、资料未知的情况包装成完整方案

### 3.2 开工检查

- 只要未确认项仍可能推翻当前方案，就不得放行
- 不得把“先出一版再说”当作放行理由

### 3.3 原型

- 原型只讲页面、流程、状态、动作和标注
- 不得把规则解释、方法论解释、契约解释铺到页面正文
- 不得让 hero、大段导语、流程总述抢占页面主体
- 若页面主体还看不出元素、入口、动作和状态承接，则视为原型失效

### 3.4 PRD

- PRD 只讲业务、规则、异常、权限、数据影响、验收
- 不得把 OMP 自己如何分工、如何生成、如何承接写进正式正文

### 3.5 评审

- 评审发现样例污染真实、未确认伪装已确认、脱锚扩写、元话语污染正式产物时，至少记为重要项
- 若问题已影响范围、版本或核心判断，直接记为阻断项

## 4. 回退规则

出现以下情况时，必须回退，不得硬推：

- 样例假设混入真实协作结论
- 未确认事实被当成稳定事实
- 当前产物与核心锚点失配
- 正式产物正文出现明显方法论元话语

回退方向：

- 可直接修正文案或结构：`internal_repair`
- 会影响当前版本方案：`reopen_alignment`
- 缺关键材料：`need_materials`

## 5. 适用范围

本契约优先约束以下动作：

- `omp-reply`
- `omp-check`
- `omp-ready`
- `omp-proto`
- `omp-prd`
- `omp-review`
