# OhMyPm 上下文防爆规则

## 1. 目标

当输入材料、当前上下文或待生成内容过大时，必须主动改写处理方式，避免上下文爆炸导致降智、遗漏或输出质量下降。

## 2. 风险来源

- 单次输入材料过长
- 多轮对齐累计上下文过长
- 单次输出目标过长
- 同时引用过多上游产物
- 长文生成后未摘要回收

## 3. 强制规则

- 不得把超长材料直接整篇载入上下文
- 不得在高风险状态下一次性输出整份长文
- 长文生成后必须做摘要回收与索引保留

## 4. 处理策略

- 输入分块：先抽取结构、事实、问题、结论
- 输出分块：按页面、模块、章节、版本差异逐段生成
- 生成后回收：只保留摘要、索引和稳定路径
- 引用裁剪：每轮只保留当前任务必要证据

## 5. 适用对象

- 回应摘要
- 长版方案
- 交付型原型说明
- PRD
- 评审材料包

## 6. 输入分块协议

当来源材料触发防爆时，按以下顺序处理：

1. 先判断是否已有缓存提取结果
2. 若无缓存，生成结构化提取
3. 只把提取结果载入当前上下文
4. 仅在提取结果不足时，定向回查原文局部片段

默认缓存路径：

- `docs/ohmypm/cache/material-extract.md`

结构化提取至少包含：

- 业务目标与背景
- 角色与职责
- 关键流程
- 规则与约束
- 数据与接口线索
- 异常与边界
- 待分类内容

## 7. 输出分块协议

当目标输出触发防爆时，按以下顺序处理：

1. 先生成分块计划
2. 再按块逐段生成
3. 每块生成后写出摘要
4. 全部完成后只保留摘要、索引和稳定路径

常见分块方式：

- 回应稿：按问题域或模块分块
- 原型说明：按页面分块
- PRD：按章节或页面单元分块
- 评审材料：按变化点、风险点、待决策点分块

## 8. 风险等级

上下文风险固定分为三档：

- `low`
- `medium`
- `high`

当风险为 `high` 时，不得继续整篇输入或整篇输出。
