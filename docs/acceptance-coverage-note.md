# 验收覆盖补充说明

## 1. 本说明解决什么问题
- 这份说明用于回答“本轮审批路径扩展做完以后，到底要验哪些点，才能证明它只是原能力增强而不是留下隐性缺口”。

## 2. 当前验收覆盖结论
- 当前验收覆盖还不完整，主要缺的是新增审批路径扩展带来的关键场景覆盖。
- 后续不是简单补一句“支持新增路径”就够，而是要把主流程、分支流程、异常流程和联动影响都补到验收范围里。

## 3. 至少应覆盖的验收点

### 3.1 主流程验收
- 新增审批路径能够按预期进入、流转、结束。
- 关键节点的页面展示、处理动作和状态变化符合当前版本方案。

### 3.2 分支流程验收
- 不同审批分支的进入条件清晰且执行结果正确。
- 新增分支不会误伤原有审批路径。

### 3.3 异常流程验收
- 驳回、退回、重新提交、流程中断等异常情况仍能按规则处理。
- 异常情况下的状态、记录和通知结果可被清楚验证。

### 3.4 联动影响验收
- 通知是否按预期触发或不触发。
- 查询、列表、详情、记录等受影响位置是否同步更新。
- 统计或状态汇总口径若受影响，结果应一致。

### 3.5 兼容性验收
- 既有审批路径在本轮改动后仍能正常使用。
- 不新增错误入口、不打断原有处理习惯、不造成旧数据解释混乱。

## 4. 当前建议口径
- 对外说明时，应强调“新增路径可用”与“原路径不受破坏”必须同时成立。
- 不要只写新增分支本身，要把联动位置和异常情况一起纳入验收表达。

## 5. 进入更重阶段前至少要补齐的内容
- 一版主流程与分支流程验收点
- 一版异常流程验收点
- 一版通知、记录、查询结果相关联动验收点
- 一版兼容性验收点
