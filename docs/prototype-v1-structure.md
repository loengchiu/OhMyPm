# 第一版原型结构稿

## 1. 使用方式
- 这份结构稿用于指导第一版交付型原型的页面组织和编号标注。
- 原型正文只展示页面结构、流程关系和编号入口，不铺长段规则说明。
- 详细规则、异常、验收口径继续由 PRD 承接。

## 2. 页面结构

### P1 审批入口与待办列表页
页面目标：
- 让评审和研发先看到审批能力从哪里进入，以及新增审批路径在入口层如何体现。

页面结构：
- 入口区
- 查询筛选区
- 待办/已办列表区
- 状态标签区
- 快速进入详情动作区

建议标注：
- `01` 审批入口位置
- `02` 新增审批路径触发条件
- `03` 列表状态标签变化
- `04` 列表结果受状态变化影响的位置

与其他页面关系：
- 从本页进入 `P2 审批详情与处理页`
- 本页状态结果需要与 `P3 流转记录视图`、`P4 通知触达视图` 保持一致

### P2 审批详情与处理页
页面目标：
- 作为主流程和关键分支的核心承载页。

页面结构：
- 审批对象信息区
- 当前状态区
- 审批节点区
- 可执行动作区
- 分支说明区
- 编号标注区

建议标注：
- `05` 当前审批节点
- `06` 新增审批分支出现条件
- `07` 主流程处理动作
- `08` 结果判断分支
- `09` 状态变化点
- `10` 需要在 PRD 中补正式规则的位置

与其他页面关系：
- 从 `P1` 进入本页
- 本页动作结果同步到 `P3` 和 `P4`
- 异常动作从本页分流到异常处理说明

### P3 审批流转记录 / 轨迹视图
页面目标：
- 把流程留痕、关键节点变化和异常记录讲清楚。

页面结构：
- 流程轨迹区
- 节点记录区
- 异常记录区
- 结果追溯区

建议标注：
- `11` 关键节点进入记录
- `12` 处理动作结果记录
- `13` 驳回 / 退回 / 重新提交记录
- `14` 特殊分支进入记录

与其他页面关系：
- 承接 `P2` 中的动作结果
- 用于核对状态变化是否与 `P1`、`P4` 一致

### P4 通知触达视图
页面目标：
- 把新增审批路径下的通知联动讲清楚。

页面结构：
- 通知触发节点区
- 通知对象区
- 触发 / 不触发条件区
- 结果校验区

建议标注：
- `15` 通知触发点
- `16` 通知对象
- `17` 不触发条件
- `18` 通知结果与状态结果一致性校验点

与其他页面关系：
- 由 `P2` 中的动作和状态变化触发
- 结果需与 `P1` 列表状态和 `P3` 留痕记录对齐

## 3. 流程结构

### F1 主流程
流程链：
- `P1 入口进入`
- `P2 查看详情并处理`
- `P2 经过新增审批路径主链路`
- `P3 留痕记录`
- `P4 触发通知`
- `P1 / P2 展示结果状态`

重点标注：
- `06` 新增路径进入条件
- `07` 主流程处理动作
- `09` 状态变化点
- `15` 通知触发点

### F2 关键分支流程
流程链：
- `P2 命中条件分支`
- 按条件进入原有路径或新增路径
- 若结果判断不同，则同步更新 `P3` 和 `P4`

重点标注：
- `08` 结果判断分支
- `14` 特殊分支进入记录
- `18` 联动一致性校验点

### F3 异常流程
流程链：
- `P2 发起驳回 / 退回 / 重新提交 / 流程中断`
- `P3 写入异常记录`
- 必要时 `P4` 展示通知结果
- 返回 `P1` 或回到 `P2`

重点标注：
- `13` 异常记录
- `17` 不触发条件

### F4 联动流程
流程链：
- `P2` 动作触发状态变化
- `P1` 列表状态更新
- `P3` 留痕更新
- `P4` 通知结果更新

重点标注：
- `03` 列表状态标签变化
- `09` 状态变化点
- `12` 动作结果记录
- `18` 一致性校验点

## 4. 原型出图顺序建议
1. 先画 `P2 审批详情与处理页`，因为它是主流程核心。
2. 再补 `P1 审批入口与待办列表页`，把入口与结果状态接上。
3. 然后补 `P3 流转记录视图`，把留痕链路补齐。
4. 最后补 `P4 通知触达视图`，把联动结果补齐。

## 5. 当前结论
- 第一版原型已经不只是页面清单，而是具备了按页展开、按流程串联、按编号标注的结构基础。
- 后续若继续推进，下一步可以直接开始按这份结构稿绘制或描述具体原型页面。
