# 验收覆盖正式承接稿

## 1. 适用范围
- 本稿适用于当前版本内的审批路径扩展。
- 本稿用于正式承接稳定对齐包中的验收覆盖结论，作为后续 PRD 中“验收与交付说明”的直接基础。

## 2. 总体验收结论
- 本轮验收不能只证明“新增路径可用”。
- 本轮验收还必须同时证明“原路径未被破坏”“联动结果仍正确”“异常情况下结果仍可控”。

## 3. 主流程验收
- 新增审批路径能够按预期进入、流转、结束。
- 关键节点的页面展示、处理动作和状态变化符合当前版本方案。
- 主流程中不会出现错误入口、错误跳转或错误结束状态。

## 4. 分支流程验收
- 不同审批分支的进入条件清晰且执行结果正确。
- 新增分支不会误伤原有审批路径。
- 分支切换、分支回退或条件判断变化后，结果仍与规则说明一致。

## 5. 异常流程验收
- 驳回、退回、重新提交、流程中断等异常情况仍能按规则处理。
- 异常情况下的状态变化、记录结果和通知结果可被验证。
- 异常处理后不会留下无法解释的状态或记录缺口。

## 6. 联动影响验收
- 通知是否按预期触发或不触发。
- 查询、列表、详情、记录等受影响位置是否同步更新。
- 统计或状态汇总口径若受影响，结果应保持一致。

## 7. 兼容性验收
- 既有审批路径在本轮改动后仍能正常使用。
- 不新增错误入口，不打断原有处理习惯，不造成旧数据解释混乱。
- 原有规则在未被明确修改的范围内继续有效。

## 8. 正式材料使用要求
- 原型负责标出需要重点验证的关键路径、关键状态和关键页面位置。
- PRD 负责把本稿内容整理为正式验收标准和交付说明。
- 若后续正式材料与本稿冲突，应先回到对齐层修订，不得各自保留不同版本。
