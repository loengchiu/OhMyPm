## 2. 背景与目标

资产管理模块面向智慧服务区物业管理场景，围绕“标准物资目录 + 服务区实际台账”建立统一资产管理能力。本期要解决的问题，不是单一单据流转，而是让标准物资维护、服务区资产入账、异动扣减、盘点回写和统计分析形成一套数量闭环。

本期目标：
- 建立运营部维护的资产库标准物资目录，作为所有资产业务的统一物资来源。
- 建立服务区资产台账，让每个服务区都能维护本区实际数量、编码状态、在库状态和归档状态。
- 打通入库、出库、调拨、报废、盘点五类核心业务，确保业务完成后统一回写台账。
- 打通统计分析，使统计结果与台账和异动结果保持一致。

本期不解决：
- 资产报损能力
- 外部财务系统自动回传编码或自动记账接口
- 完整的逐件唯一编码重资产单体模型
- 折旧计算
- 预警和提醒能力

## 3. 本次范围

本次范围包括：
- 资产库管理
- 服务区资产台账管理
- 入库业务
- 出库业务
- 调拨业务
- 报废业务
- 盘点任务与盘点详情
- 资产统计分析

本次范围不包括：
- 与资产模块无关的其他物业业务页面
- 超出本期边界的外部系统自动集成
- 脱离双层模型的独立重资产精细化单体资产管理

## 4. 整体方案概述

### 4.1 模块边界

模块采用双层模型：
- 资产库：由运营部维护标准物资目录，负责定义“有什么物资”。
- 服务区资产台账：由服务区管理员维护本区实际持有资产，负责记录“本区有多少、是否可用、是否补录编码、是否归档”。

所有业务动作都围绕这两层展开：
- 入库从资产库选择标准物资，把数量记入服务区台账。
- 出库、调拨、报废从服务区台账出发，完成后回写台账。
- 盘点以服务区台账为账面基础，差异确认后回写台账。
- 统计以台账、异动结果和盘点结果为统一统计口径。

### 4.2 流程概述

主流程如下：
1. 运营部维护标准物资目录。
2. 服务区或管理中心发起入库，符合条件时进入财务审核，确认后资产进入服务区台账。
3. 服务区从台账发起出库、调拨、报废等异动。
4. 异动完成后统一回写台账数量、状态和履历。
5. 运营部或区域中心发起盘点任务，服务区执行盘点并确认差异。
6. 统计页汇总台账、异动和盘点结果。

### 4.3 与存量能力的关系

本模块不是新增独立系统，而是在现有物业管理系统内新增资产业务能力。系统需要复用现有角色体系、组织范围、导出方式和基础枚举管理方式，同时确保服务区级别的数据隔离和管理中心级别的全局查看能力。

## 6. 业务规则说明

### 6.1 资产库与台账基础规则

- 标准物资由运营部维护，是资产业务的统一物资来源。
- 服务区资产台账必须关联标准物资，不允许脱离标准目录直接新增独立资产记录。
- 台账至少管理以下核心结果字段：当前数量、可用数量、在库状态、编码状态、归档状态。
- 当台账数量扣减为 0 时，记录应默认归档，不在默认列表中展示。
- 台账详情应能查看履历，补录编码后应更新编码状态，并允许已补录编码的记录进入标签打印能力。

### 6.2 入库规则

- 入库可由服务区发起，也可由运营部或区域中心代发起。
- 入库明细必须基于标准物资选择，填写数量和税后单价，系统自动计算明细税后价值。
- 当任一入库明细税后价值大于 2000 元时，入库单进入“待财务审核”。
- 财务审核通过后，状态进入“待确认”；财务驳回后，状态进入“已驳回”。
- 服务区确认入库后，目标服务区台账数量增加，在库状态默认为“闲置”。

### 6.3 出库规则

- 出库从服务区台账出发，只能选择本服务区可用数量大于 0 的台账记录。
- 出库申请必须填写责任人、使用部门、使用人。
- 当任一出库明细税后价值大于 2000 元时，出库单进入“待财务审核”；否则直接进入“待确认”。
- 服务区确认出库后，台账数量扣减，并沉淀领用结果状态“正常使用”。
- 若服务区拒绝出库，状态应更新为“已驳回”。

### 6.4 调拨规则

- 调拨从调出服务区台账出发，调拨数量不能超过调出服务区可用数量。
- 调拨先进入“审核中”，由运营部、区域中心或具备权限的角色审核。
- 审核通过后进入“待确认”，由调入服务区管理员确认接收。
- 调入确认后，调出服务区台账数量扣减，调入服务区台账数量增加。
- 调入方拒绝接收时，状态更新为“已驳回”。

### 6.5 报废规则

- 报废从服务区台账发起，必须上传证明材料。
- 报废数量不能超过可用数量。
- 报废申请进入“待审核”，由运营部、区域中心或财务部审核。
- 审核通过后，台账数量扣减，系统生成报废履历。
- 数量扣减为 0 时，应触发归档状态更新。

### 6.6 盘点规则

- 盘点任务由运营部或区域中心发起，按“服务区 + 标准物资”粒度生成盘点明细。
- 盘点明细带出账面数量和编码状态。
- 未补录编码的资产采用数量盘点方式；已补录或部分补录的资产采用编码盘点方式。
- 系统根据实盘数量计算差异数量和差异类型。
- 差异明细必须填写复核意见，并选择确认盘盈、确认盘亏或忽略差异。
- 只有全部差异处理完成后，盘点任务才允许进入“已完成”。
- 盘点完成后，应统一回写台账数量、编码状态和履历记录。

### 6.7 统计规则

- 统计页按时间范围、服务区、物资分类、业务类型等口径查询。
- 台账总览至少包括总数量、分类结构占比、编码覆盖率、总金额。
- 入出调拨统计应支持按月、按季、按年切换趋势。
- 盘点结果统计至少包括盘盈数量、盘亏数量、差异率、已处理率。
- 点击图表数据点时，应能钻取到对应业务记录列表。
- 统计结果必须与台账、异动结果和盘点回写结果保持一致。

## 7. 异常与边界说明

### 7.1 异常类型

- 财务审核驳回
- 运营部 / 区域中心审核驳回
- 服务区确认拒绝
- 调入方拒绝接收
- 报废审核驳回
- 盘点存在未确认差异
- 导入文件格式错误或数据校验失败

### 7.2 回退规则

- 财务审核驳回后，单据状态更新为“已驳回”，不回写台账数量。
- 服务区拒绝入库或出库时，单据状态更新为“已驳回”，不执行数量回写。
- 调拨审核驳回或调入方拒绝接收时，调拨流程终止，不回写任何一侧数量。
- 报废审核驳回后，不扣减台账数量。
- 盘点存在未确认差异时，不允许完成任务，也不允许回写台账。
- 导入失败时，应展示失败原因，并保持当前表单或任务结果不被错误覆盖。

### 7.3 明确边界

- 本期不引入报损流程。
- 本期不要求外部财务系统自动回传编码。
- 本期不建立逐件唯一编码驱动的完整单体资产模型。
- 本期不做资产折旧和预警。
- 标签打印只基于已补录编码的台账记录。

## 8. 角色、权限与数据影响

### 8.1 角色清单

- 运营部管理员
- 服务区管理员
- 区域中心管理员
- 财务部管理员

### 8.2 角色动作矩阵

| 角色 | 关键动作 |
| --- | --- |
| 运营部管理员 | 维护资产库、代发起入库、审核调拨、审核报废、查看全局统计 |
| 服务区管理员 | 维护本区台账、补录编码、打印标签、发起出库/调拨/报废、确认入库/出库、执行盘点 |
| 区域中心管理员 | 跟踪所辖服务区业务、审核调拨和报废、发起盘点、确认盘点结果 |
| 财务部管理员 | 审核高价值入库和出库、查看统计数据 |

### 8.3 数据影响

- 资产库增加标准物资主数据。
- 服务区台账承接数量、可用数量、编码状态、在库状态、归档状态。
- 入库、出库、调拨、报废、盘点均生成对应业务单据与明细。
- 异动完成后，系统生成履历记录和审批记录。
- 统计页汇总台账、异动和盘点结果。

### 8.4 查询、通知、记录影响

- 列表查询统一支持状态、服务区、关键字、时间范围等筛选。
- 导入、导出结果需要形成可追踪的成功 / 失败反馈。
- 履历记录需要覆盖入库、出库、调拨入、调拨出、报废、盘点调整。
- 审批记录至少记录审批人、审批时间、审批动作、审批意见。

## 9. 验收与交付说明

### 9.1 主流程验收

- 资产库能维护标准物资，且已引用物资不可直接删除。
- 服务区台账能准确反映本区资产数量、编码状态和归档状态。
- 入库确认后，目标服务区台账数量增加。
- 出库、调拨、报废完成后，台账数量按规则扣减。
- 调拨完成后，调出与调入两侧台账数量同时正确回写。
- 盘点完成后，差异结果正确回写台账。

### 9.2 分支流程验收

- 入库和出库在任一明细税后价值大于 2000 元时进入财务审核。
- 调拨在审核通过前不得进入调入确认。
- 报废必须上传证明材料才能提交。
- 已补录编码的台账记录才能进入标签打印。

### 9.3 异常流程验收

- 审核驳回后不回写数量。
- 服务区拒绝确认后单据正确终止。
- 导入文件格式错误时给出明确提示，不污染现有数据。
- 盘点存在未确认差异时不能完成任务。

### 9.4 联动影响验收

- 异动结果与台账数量一致。
- 台账、履历、审批记录和统计页口径一致。
- 统计页下钻结果与业务记录一致。
- 台账数量清零后自动归档，不在默认列表中展示。
